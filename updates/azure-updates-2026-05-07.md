2026年5月初旬に、Azure SQL Managed InstanceのBusiness Criticalサービス層に対して、メモリの適正化（Right-size memory）機能のアップデートがパブリックプレビューとして提供されました。このアップデートの背景には、パフォーマンス向上とリソースの最適利用という目的があります。従来、Business Critical層のインスタンスでは、ワークロードの変動に対応するために必要以上のメモリを割り当てる、いわゆるオーバープロビジョニングが発生しやすい状況がありました。これにより、コスト効率の低下やリソースの無駄遣いが課題となっていました。

今回のアップデートでは、インスタンスに割り当てるメモリ量をワークロードに応じて適切なサイズに調整する機能が強化されています。これにより、過剰なメモリ割り当てを防ぎつつ、必要なパフォーマンスを確保することが可能となります。技術的には、Azure SQL Managed Instanceのリソース管理層が、ワークロードの実行状況やリソース消費状況を監視し、最適なメモリ割り当てを自動的に適用する仕組みが導入されています。これにより、ユーザーは手動でのリソース調整作業を減らしつつ、安定したパフォーマンスを享受できます。

この機能は、特にワークロードの変動が大きい業務システムや、コスト効率を重視するシナリオで有効です。たとえば、ピーク時とアイドル時でリソース消費が大きく異なるバッチ処理や、季節変動のある業務アプリケーションなどで活用が期待されます。利用にあたっては、Business Critical層のAzure SQL Managed Instanceを利用していることが前提となります。また、パブリックプレビュー段階での提供であるため、運用環境での利用には十分な検証が推奨されます。

本機能は、他のAzureサービス、たとえばAzure MonitorやAzure Advisorなどと連携することで、リソース使用状況の可視化や最適化の提案を受けることが可能です。これにより、より高度な運用管理やコスト最適化が実現できます。なお、詳細な実装方法や制限事項については、公式ドキュメントやアップデート情報（https://azure.microsoft.com/updates?id=560283）を参照することが推奨されます。

**詳細**:

Azure Backupにおいて、「Bulk Restore for Azure Virtual Machines」のパブリックプレビューが開始されました。本アップデートの背景には、大規模な障害対応やリカバリー作業時に複数の仮想マシン（VM）の同時復元が求められるケースが増加していることがあります。従来はVMごとに個別でリストア作業を行う必要があり、運用負荷や作業時間の増大が課題となっていました。今回のアップデートにより、最大100台までのAzure仮想マシンを一度の操作でまとめてリストアできる「Bulk Restore」機能が追加され、復元作業の効率化が実現されています。

具体的な機能としては、Azure Backupの管理画面から複数のVMを選択し、まとめてリストアジョブを実行することが可能です。これにより、複数VMの同時復元が必要な大規模障害時や、テスト環境の一括復元などのシナリオで、作業の自動化・省力化が図れます。各VMごとにリストア先や設定を個別に指定できるため、柔軟性と制御性も維持されています。

技術的な仕組みとしては、Azure Backupのバックアップデータを用いて、選択した複数VMに対して並列的にリストア処理を実行します。これにより、従来のシリアルなリストア処理に比べて大幅な時間短縮が期待できます。操作はAzureポータル上でGUIベースで行うことができ、PowerShellやCLIによる自動化も今後の展開が期待されます。

活用シナリオとしては、データセンター全体の障害復旧、開発・テスト環境の一括復元、複数VMを含むサービスのロールバックなどが挙げられます。特に、災害対策（DR: Disaster Recovery）や大規模なパッチ適用後のリカバリーなど、迅速な復旧が求められる場面で有効です。

注意点としては、現時点で最大100台までのVMが対象であること、パブリックプレビュー段階であるため本番環境での利用には慎重な検証が必要であることが挙げられます。また、リストア対象となるVMのバックアップがAzure Backupで取得されている必要があります。

関連するAzureサービスとしては、Azure Virtual MachinesおよびAzure Backupが中心となりますが、復元後のネットワーク設定やストレージ構成など、他のAzureリソースとの連携も考慮する必要があります。今回のアップデートにより、Azure上での大規模なVMリストア作業がより効率的かつ柔軟に実施可能となります。

注意点としては、現在パブリックプレビュー段階であるため、機能や安定性に制限がある場合があります。正式リリース前のため、運用環境での利用には十分な検証が必要です。また、MCPによるエージェント機能の動作や、既存のAzure Cosmos DBツールとの互換性についても、最新のドキュメントを参照しながら利用することが推奨されます。

関連するAzureサービスとの連携については、Azure Cosmos DB ShellはAzure Cosmos DBのリソース管理に特化したツールですが、Azure CLIやAzure Portalなど他の管理ツールと併用することで、より柔軟な運用が可能です。今後、他サービスとの連携や拡張も期待されますが、現時点ではAzure Cosmos DBのデータワークフローに特化したCLIとして位置付けられています。
