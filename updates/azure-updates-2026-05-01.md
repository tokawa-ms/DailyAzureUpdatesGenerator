**カテゴリ**: Launched, Storage, Azure Blob Storage, Features

**要約**:

【何が更新されたか】  
Azure Blob Storageにおいて、User Delegation SAS（Shared Access Signature）のプレフィックススコープアクセスが全Azureリージョンで一般提供（GA）されました。

【主な変更点や新機能】  
これまでSASトークンはコンテナ単位または個々のBlob単位でアクセス範囲を指定できましたが、今回のアップデートにより、Blob名のプレフィックス（接頭辞）を指定してアクセス範囲を細かく制御できるようになりました。これにより、特定のパターンに一致する複数のBlobへのアクセス許可を柔軟に設定できます。

【影響を受ける対象】  
Azure Blob Storageを利用している技術者や開発者、特にSASによるアクセス制御を実装しているシステムが対象となります。

【注意点】  
プレフィックススコープアクセスを利用する際は、SASトークンの発行と管理において、アクセス範囲の設定ミスによる情報漏洩リスクに注意が必要です。既存のSAS運用に影響がないか、事前に検証を行うことを推奨します。

**詳細**:

本アップデートは、Azure Blob StorageにおけるUser Delegation SAS（Shared Access Signature）に対して、プレフィックス単位でのアクセス許可を付与する機能が一般提供（GA）となったことを示しています。これまでBlob StorageのSASトークンは、コンテナ全体または個々のBlob単位でのアクセススコープのみをサポートしていましたが、本リリースにより、Blob名のプレフィックスを指定してアクセス範囲を限定することが可能となりました。

この機能の主な目的は、より細かなアクセス制御を実現することにあります。従来のコンテナスコープでは、コンテナ内の全てのBlobに対してアクセス権が付与されてしまうため、最小権限の原則に基づいた運用が難しいケースがありました。一方、個別Blobスコープでは、多数のBlobにアクセス権を付与したい場合にトークンの発行や管理が煩雑になる問題がありました。プレフィックススコープの導入により、特定の名前パターンに一致する複数のBlobに対して一括でアクセス権を付与できるため、セキュリティと運用効率の両立が図れます。

技術的には、User Delegation SASの発行時に、アクセス対象となるBlobの名前に共通するプレフィックスを指定することで、そのプレフィックスに一致する全てのBlobへのアクセスが許可されます。これにより、例えば「logs/2024/」というプレフィックスを指定した場合、logs/2024/で始まる全てのBlobに対して一つのSASトークンでアクセス可能となります。SASトークンの生成自体はAzure AD認証を用いたUser Delegation SASの標準的な手順に従い、Azure Storage SDKやREST APIを通じて実施します。

活用シナリオとしては、特定のプロジェクトや期間ごとに格納されたBlob群への一時的なアクセス権付与、またはアプリケーションやユーザーごとにディレクトリ風のプレフィックスで管理されたデータへの限定的なアクセス許可などが挙げられます。これにより、不要な権限の付与を避けつつ、柔軟なアクセス制御が可能となります。

注意点としては、プレフィックススコープのSASトークンを発行する際には、指定したプレフィックスに一致する全てのBlobが対象となるため、意図しないデータへのアクセスが許可されないよう、プレフィックスの設計や運用管理に十分注意する必要があります。また、SASトークンの有効期限やアクセス許可（読み取り、書き込み、削除など）も従来通り慎重に設定することが推奨されます。

本機能はAzure Blob StorageのUser Delegation SASに対する拡張であり、他のAzure Storageサービス（例：Azure FilesやAzure Queues）には直接適用されません。Blob Storageを利用したデータ共有やアクセス制御の高度化を図る際に有用な機能となります。
